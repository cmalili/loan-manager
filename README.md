# Loan Manager

Loan Manager is a scaffold for a full-stack application for managing loans, borrowers, repayments, and related operational workflows.

The repository is currently in an early setup phase. It contains the initial project structure for separate frontend and backend workstreams, plus a documentation folder for requirements and planning artifacts.

## Repository Structure

```text
loan-manager/
├── backend/   # Backend application code
├── docs/      # Product and technical documentation
└── frontend/  # Frontend application code
```

## Current Status

- `backend/` exists but does not yet contain application code.
- `frontend/` exists but does not yet contain application code.
- `docs/requirements.md` is present as the intended home for project requirements.

## Getting Started

There is no runnable application in the repository yet. A practical next step is to define the core product requirements in [`docs/requirements.md`](/home/cmalili/loan-manager/docs/requirements.md), then scaffold the frontend and backend with the chosen stack.

Suggested setup sequence:

1. Document the product requirements and scope.
2. Choose the backend framework, database, and API approach.
3. Choose the frontend framework and UI architecture.
4. Add local development scripts and environment configuration.
5. Implement the first end-to-end loan management workflow.
